Developer & Integration Guide

KetraTerm is engineered with a strict modular boundary, decoupling core terminal mechanics, byte parsing, input encoding, and UI presentation components.

Unidirectional Data Pipeline

Unlike traditional terminal emulators where UI actions and PTY readers write asynchronously to the same grid memory, KetraTerm routes all actions along a strict, unidirectional loop.

This separation keeps GUI updates free from tearing even when the subprocess output is flooding the screen, sustaining steady 60+ FPS cycles.

Flat In-Memory Cell Storage

To eliminate memory garbage collection overhead, grid lines in the core engine use flat, parallel primitive arrays (codepoints: IntArray, attrs: LongArray) instead of wrapping cells in Java objects.

Inbound Path (Host to Screen State):
 PTY / SSH ──[Raw Bytes]──► Parser ──[Commands]──► Host ──[Mutations]──► Core Grid
                                                                           │
                                                                       [Snapshot]
                                                                           ▼
 UI Swing ◄──[Repaint]── UI Component ◄──[Lease]── Render Cache ◄── Render API

Outbound Path (User Typing to Host stdin):
 User Event ──[Key/Paste]──► Input ──[ANSI Bytes]──► Session ──► PTY / SSH

Deterministic Concurrency Architecture

KetraTerm coordinates background threads and the Swing Event Dispatch Thread (EDT) using three specialized locks inside the TerminalSession controller.

🔐

inboundLock

Guarantees that raw terminal output chunks arriving on PTY background threads are delivered to the ANSI state machine in sequential stream order.

🔐

mutationLock

Protects the grid engine. Prevents parser mutations from altering characters while the rendering thread snapshots frames, avoiding layout tearing.

🔐

outboundWriteLock

Synchronizes terminal writes. Ensures that keyboard typing, clipboard pastes, and query responses (like CPR/DSR replies) are enqueued atomically.

Monomorphized LRU Caching

For hot rendering paths, KetraTerm avoids object-boxing by using custom primitive-keyed LRU caches, including IntFontLru for codepoints font lookup and LongTextLayoutLru for pre-shaped text layouts.

How to Integrate in JVM Apps

Integrate KetraTerm easily into your Gradle projects. The workspace module manages load-time user profiles, settings persistence, and tab selections.

Implement TerminalWorkspaceListener to bind lifecycle operations (tab open, title changes, visual bell trigger) to your application panels.

Gradle Dependency Configuration

dependencies {
    // Core terminal UI widget module
    implementation("io.github.ketraterm:ketraterm-ui-swing:0.1.0")
    // Workspace, TOML config, and lifecycle manager
    implementation("io.github.ketraterm:ketraterm-workspace:0.1.0")
}

Kotlin Bootstrap Implementation

// 1. Load TOML settings from OS directories
val configManager = TerminalWorkspaceConfigManager.getDefault()
val config = configManager.load()

// 2. Create the workspace listener
val listener = object : TerminalWorkspaceListener {
    override fun tabOpened(tab: TerminalWorkspaceTab) {
        println("Session launched: ${tab.id}")
    }
    override fun tabClosed(id: String) {}
    override fun tabSelected(id: String) {}
}

// 3. Initialize the workspace state controller
val workspace = TerminalWorkspace(listener)

// 4. Define launch profile and trigger session
val shellProfile = TerminalProfile(
    name = "system-shell",
    displayName = "Default Shell",
    command = emptyList() // Falls back to system default
)
workspace.openTab(shellProfile)
